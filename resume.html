마이크로 서비스 개발

광고회사에 있으면서 외부 회사의 서버로부터 광고를 가져오는 마이크로 서비스인 adnsvc를 처음부터 설계 & 개발하였습니다. 이 서비스는 많은 네트워크 I/O bound가 예상되었기에 Golang으로 개발되고 API로는 gRPC를 사용하였습니다. 이 서비스를 개발하면서 기존 하나의 모놀리틱 서버에서 50여개의 python파일, 5000줄이 넘는 코드를 삭제할 수 있었습니다.

기존의 광고가 아닌 뉴스 기사를 읽으면 그 뉴스기사의 페이지의 수익을 나누는 리워디드 컨텐츠 모델을 지원하기 위한 서비스를 개발하였습니다. 이 서비스를 런칭하여서 아직 광고가 없는 해외 사업의 수익화 전략을 달성하는데 중요한 역할을 수행하였습니다. 빠른 서비스 런칭을 위하여 Python으로 개발되었고, gRPC를 사용했지만 http 요청도 같이 처리할 수 있도록 만들어졌습니다. 한달간 이 서비스에서 서빙한 뉴스 기사로 인한 수익은 500만원이 조금 넘습니다.

실험적인 프로젝트로 우리 제품이 아닌 곳에도 광고를 내보낼 수 있는 서버인 DSP를 개발하였습니다. 회사가 기존에 진행하던 사업이 아닌 새로운 분야로 진출하는데 기여하였습니다. 광고 서버의 한 종류인 DSP는 100ms이하의 latency를 만족시켜야 한다는 제약사항이 있어서 성능을 생각하며 개발되었습니다. DSP는 Golang으로 개발되었고, DSP의 API규격을 맞추기 위하여 HTTP API를 사용하였습니다.

안정적인 서비스 개발

테스트 코드 작성

여러가지 마이크로 서비스를 만들면서 테스트 코드를 작성하였습니다. 가장 코드가 많은 서비스인 adnsvc는 테스트 커버리지를 90%정도로 유지하고있으며, 그 외의 서비스도 80%가 넘는 테스트 커버리지로 많은 영역을 테스트하고 있습니다. adnsvc는 서비스 특성상 외부 서버의 API호출이 매우 많은데, 이를 효과적으로 테스트하기 위하여 API response 샘플을 코드와 같이 관리하여 기존에 테스트가 하나도 되지 않던 것에 비해서 많은 발전이 이루어졌습니다.

기존에 QA에서는 외부 API를 사용하는 부분을 아예 테스트하지 못했습니다. 하지만 API sample을 코드와 함께 관리하게 되면서 특정한 경우 실제 API를 보내지 않고 API response 샘플을 사용할 수 있는 기능을 개발하였습니다. 이 기능 개발로 인하여 외부 API를 사용하여 광고를 가져오는 테스트를 쉽게 할 수 있게 되었고, 전혀하지 않던 QA도 쉽게 가능하게 되었습니다.
