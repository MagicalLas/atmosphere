스프링 3주차 회고

점진적으로 개선해나간다. 회고를 하면 얻을 수 있는 좋은점입니다. 저는 이번주 어떤 것들을 발견했고, 개선했을까요?

> 발견한 것

저는 이번에 테스트 코드를 작성했어요. 간단하게 시작해나가는 방식은 좋은 점인 것 같아요. 몇가지 테스트를 작성했고, 곧 모든 테스트 케이스를 작성했죠. 나쁜 경험은 아니었어요. 그러나 저는 단순한걸 더 복잡하고 좋게 만드는 것에 두려움이 있어요. 예를들어서 AssertJ와 같은 라이브러리를 적용하는거죠. 지금 이걸 적용하면 얻는 점이 어떤건지 확신을 갖지 않는다면 적용을 꺼리게 되는 것 같아요.

왜 이렇게 꺼리는걸까를 생각해본다면, 그전까지 테스트를 많이 작성해왔고, 그 방식에 꽤 익숙해져서 새로운걸 꺼리기 때문이라는 생각이 들었어요. 새로운게 있다면 "일단 해보자"보단 "왜 해야하지"가 먼저 나오는거죠.

저는 첫주에 application service에 대한 테스트를 먼저 작성하면서 개발을 시작했고, 이번에는 테스트만 작성했는데, 오히려 첫 주에 작성한 테스트가 더 많았던 것 같아요. 테스트만 작성하는건 꽤나 답답한 느낌이 들었어요. 기존에는 테스트를 작성하는 시간이 "우리 이런걸 해야하는데, 어떻게 쓰면 좋을까?"를 생각했다면 "우리 이런 기능이 있다는데 쓰는 예시를 보여주자"에 가까웠던 것 같아요.

> 개선할 점

시간 확보가 더 필요해요. 최근에는 재택을 하고있지 않아서 출근하면 10시이고, 퇴근하면 7시 30분에서 8시가 되는 경우도 있어요. 재택을 할 때에 비해서 출퇴근에 드는 노력도 있고 절대적인 시간도 꽤나 부족한 것 같아요. 그러다보니 "일단 되게하자"가 큰 것 같아요. 첫 주는 "어떻게 하면 가장 잘할 수 있지"가 고민이었다면 지금은 "어떻게 하면 빠르게 괜찮은 수준의 결과를 낼 수 있을까"가 되버린 것 같아요. 첫 주와 같은 마음가짐을 갖기위해서 어떻게 해야할지 생각해봐야겠어요.

아니면 이게 지금의 저에게 있어서 최선일 가능성도 있는 것 같아요. 이정도보다 더 좋은 테스트를 작성하기 힘들다는거죠. 그렇네요. 다른 분들의 코드를 봐야겠어요. 첫 주는 다른 분들의 코드를 전부 보았는데, 왜 이번에는 안그랬을까요.

시간 확보와 함께 정신적으로도 조금 편해질 필요가 있는 것 같네요. 최근 많은 분들이 회사에서 퇴사를 하고있고, 제가 처음 회사에 들어갔을 때 부터 계속 같은 팀이던 분도 퇴사를 하게 되었어요. 제가 이 회사에 있는 이유가 이런 좋은 분들이 있어서 회사를 선택한 것인데, 어떻게 해야할지 고민이 되더군요. 사실 이 고민으로 토요일을 보냈어요. 토요일을 과제에 쏟았다면 더 나아졌을것이 분명해서 아쉬운 것 같아요.
